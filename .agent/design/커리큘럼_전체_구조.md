# 리듬 잉글리시 - 전체 커리큘럼 구조

## 설계 철학

### 핵심 원칙
1. **노래 중심 (Song-First)**: 각 스텝은 하나의 완성도 높은 노래입니다. 패턴 교육용이 아닌, 듣고 싶은 곡으로 만듭니다.
2. **테마 기반 (Theme-Based)**: 상황, 감정, 스토리를 중심으로 구성하여 학습자가 몰입할 수 있게 합니다.
3. **패턴 내포 (Pattern-Embedded)**: 문법 패턴은 노래 안에 자연스럽게 녹아들어 있으며, 학습 후 "사용된 표현"으로 제시됩니다.
4. **장르 다양성 (Genre Diversity)**: 각 노래는 테마에 어울리는 최적의 장르를 선택하여 음악적 완성도를 높입니다.

### 구조 개요
- **Course (코스)**: 난이도 단계 (Beginner → Elementary → Intermediate → Advanced)
- **Chapter (챕터)**: 감성 테마 + 음악 장르 (5-7 스텝으로 구성)
- **Step (스텝)**: 하나의 완성된 노래 (3-5분, 상황/감정 기반 스토리)

## 데이터 구조 연결

### Step 메타데이터
각 스텝은 다음과 같은 정보를 포함합니다:

```
- stepId: 고유 식별자
- title: 노래 제목
- story: 스토리/상황 설명
- genre: 음악 장르
- emotion: 핵심 감정
- songFile: 노래 파일명
- vocalFile: 보컬 파일명
- instrumentalFile: 반주 파일명
- lyricsFile: 가사 JSON 파일명
- usedPatterns: [사용된 패턴 ID 배열] - 노래 생성 후 태깅
- difficulty: 난이도 (1-5)
- unlocked: 잠금 여부
```

### 패턴 연결 방식
- 노래 제작 시: 테마와 스토리에 집중
- 노래 완성 후: 가사 분석을 통해 사용된 패턴들을 `usedPatterns`에 태깅
- 학습 시: 노래를 먼저 즐기고, "Learn" 모드에서 사용된 패턴 설명 제공
